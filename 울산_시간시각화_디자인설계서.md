# 울산 버스 시간축 시각화 — 디자인 재설계서
ulsan_time_anim 재디자인 / 2026-04-07 데이터 기준

## 1. 핵심 컨셉: "시간이 밝기를 만든다"

라이트/다크를 사용자가 따로 누르는 토글로 두지 않는다. **애니메이션 속의 현재 시각(時)이 곧 화면의 밝기**가 된다.

- 밤 → 화면이 어두운 남색, 인구 밀집지는 "불을 켠 듯" 빛난다.
- 낮 → 화면이 흰색으로 밝아지고, 인구 밀집은 검정 명암(잉크)으로 표현된다.
- 여명(일출)·낙조(일몰) → 약 1시간에 걸쳐 두 모드가 서서히 교체되며, 해가 뜨고 지는 느낌을 준다.

즉 "낮/밤 모드"는 시간축 재생과 한 몸이다. 별도의 수동 라이트/다크는 보조 장치로만 둔다(§6).

기준 시각(울산 2026-04-07): 일출 ≈ 05:50, 일몰 ≈ 18:55.

---

## 2. 인구 분포 표현 (배경 레이어, 100m 격자)

밀도값 `t = log(1+인구) / log(1+상한)` (0~1)을 쓰고, 모드에 따라 색을 다르게 입힌다.

### 2-1. 밤 (불빛 모드)
- 배경: 짙은 남색 `#0A1430`.
- 격자 셀: 저밀도는 배경에 묻히는 어두운 남색, 고밀도로 갈수록 **따뜻한 흰빛(도시 야경)**.
  - 저(t≈0): `#0E1A3A` (거의 배경)
  - 중(t≈0.5): `#5A6FB0` (옅은 푸른 빛)
  - 고(t≈1): `#FFF3D0` (불 켜진 흰/미색)
- 의미: 밤에 사람이 모여 사는 곳일수록 환하게 빛난다.

### 2-2. 낮 (잉크 명암 모드)
- 배경: 밝은 흰색 `#F4F6FA`.
- 격자 셀: 저밀도는 옅은 회색, 고밀도는 진한 먹색(검정).
  - 저(t≈0): `#E4E8F0` (배경에 가까운 옅은 회청)
  - 중(t≈0.5): `#8A93A6`
  - 고(t≈1): `#1A2030` (진한 먹/검정)
- 의미: 낮엔 밝은 종이에 검정 먹으로 밀도를 그린 듯.

### 2-3. 여명·낙조 (전환 + 하늘색)
- 전환 구간 동안 위 두 색을 보간하고, 하늘빛 틴트를 더한다.
  - 여명: 푸른 새벽빛(`#3A5AA0`) → 점차 흰색.
  - 낙조: 흰 낮 → 주황 노을(`#E8853A`) → 남색 밤.
- 인구 셀은 전환과 함께 "불이 켜지거나 꺼지는" 중간 단계를 거친다(밤 불빛 ↔ 낮 잉크).

## 3. 시간대별 색 전환 로직 (1시간 페이드)

낮 정도 `d` (0=완전한 밤, 1=완전한 낮)를 시각으로 계산한다.

- 일출 전후 **±30분(총 1시간)** 동안 d: 0 → 1 로 부드럽게(smoothstep) 증가.
- 일몰 전후 **±30분(총 1시간)** 동안 d: 1 → 0 로 감소.
- 그 외 낮은 d=1, 밤은 d=0.

색 합성:
```
배경색   = lerp(밤배경, 낮배경, d)  + 여명/낙조 하늘 틴트(전환대에서만)
인구 셀색 = lerp(밤셀색(t), 낮셀색(t), d)
```
- smoothstep(부드러운 가감속)으로 "해가 천천히 넘어가는" 느낌.
- 전환대(여명·낙조)에서만 하늘 틴트 가중치를 종 모양(0→1→0)으로 추가.

권장 파라미터: 일출 05:50 / 일몰 18:55, 전환 폭 ±30분. (조정 가능)

## 4. 버스 승객(노선) 마크: 색 사각형, 면적 = 인원

원은 반지름↔면적 관계가 직관적이지 않아 비율 비교가 어렵다. **사각형의 면적**으로 인원을 표현한다.

- 마크 = 정사각형. **한 변 길이 `s = k · √(인원)`** → 면적(s²)이 인원에 정비례.
  - 예) 인원 4명은 1명보다 변 길이 2배, 면적 4배.
- 색 = 노선별 고유색(황금각 분산). 같은 노선은 같은 색.
- 위치 = 그 시각 운행 중 승객이 승차→하차 경유정류장 경로를 따라 이동(기존 로직 유지).
- 크기 기준값(권장): `k`는 1명일 때 한 변 ≈ 3px, 상한 ≈ 22px(과대 방지). min/max 클램프.
- 가독성:
  - 밤: 사각형은 채도 높은 색 + 약한 외곽 글로우(야경과 어울림).
  - 낮: 사각형에 얇은 어두운 외곽선(흰 배경에서 분리), 채도 유지.
- 범례에 "사각형 면적 = 인원" 척도(예: 1·5·20·50명 사각형)를 둔다.

선택지: '노선 탑승 인원' vs '정류장 승차 인원' 중 어느 것을 면적에 매핑할지 토글로 둘 수 있음(기본=정류장 승차 인원, 데이터 100% 신뢰).

## 5. 화면 구성

- 상단 띠: 하루 시간 그라데이션 + 해/달 마커(현재 시각 위치). 시간 표현 유지.
- 우상단: 시계(HH:MM) + 현재 단계(🌅여명/☀낮/🌇낙조/🌑밤) + 운행 중 인원.
- 좌측: 범례 — ① 인구밀도(밤/낮 두 색 척도 동시 표기) ② 사각형 면적=인원 척도 ③ 노선색 안내.
- 하단: 재생/정지·속도·시간 스크럽바·시간대별 승차 히스토그램.
- 이용자유형 필터(전체/일반/어린이/청소년/경로/장애/국가유공) 유지.

## 6. 라이트/다크 모드 운영 방안 (권장안)

"시간 구동"이 기본이지만, 정지 화면 캡처·발표·접근성을 위해 강제 모드도 필요하다. 3안 비교:

| 안 | 동작 | 장점 | 단점 |
|---|---|---|---|
| A. 시간 자동 | 시각이 밝기를 결정(토글 없음) | 컨셉과 완전 일치, 자연스러움 | 특정 밝기로 고정 캡처 불가 |
| B. 수동 토글만 | 사용자가 낮/밤 고정 | 캡처·취향 대응 | 시간 흐름의 주야 연출이 사라짐 |
| **C. 자동 + 강제(추천)** | 기본 자동, 버튼으로 '자동/낮 고정/밤 고정' 선택 | 둘 다 만족 | 구현 약간 복잡 |

**권장: C안.** 우상단/상단에 `자동 · 낮 · 밤` 3단 버튼.
- 자동(기본): §3 로직대로 시각이 밝기 결정.
- 낮 고정: d=1 강제(흰 배경·검정 명암). 발표·인쇄용.
- 밤 고정: d=0 강제(남색·불빛). 야경 강조용.

이렇게 하면 "라이트/다크"는 곧 "낮/밤"이고, 자동이면 시간이, 수동이면 사용자가 정한다 — 별도 개념을 만들 필요가 없다.

## 7. 데이터·구현 노트

- 데이터: card_records(01) + _etc(02~06) 병합, 정류장 승차/노선 경로는 기존 파이프라인 유지(좌표 오매칭 스파이크 제거 포함).
- 성능: 사각형 fillRect는 원(arc)보다 빠름 → 마크 수 늘려도 유리.
- 색 보간은 매 프레임 d, t로 계산(셀 8,029개 + 활성 마크 수천 → 60fps 가능).
- 확대/이동(휠·드래그·더블클릭 초기화) 유지.
- 인구 상한은 99퍼센타일로 고정(이상치 완화).
