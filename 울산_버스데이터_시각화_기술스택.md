# 울산 버스 인구이동 시각화 — 기술 스택 문서

> 작성일: 2026-06-15
> 관련 문서: `울산_버스데이터_시각화_자료정리.md` (설계서)
> 목적: 인구밀도 배경 + 버스/승객 이동 흐름을 하루 시간축·주야 연출로 시각화하는 웹 애플리케이션의 기술 구성 정의

---

## 0. 전체 아키텍처 개요

```
[공공데이터 API]          [데이터 파이프라인]            [프론트엔드 웹]
 국토부 교통카드      →    Python 수집·전처리      →    deck.gl + MapLibre
 울산 BIS                 좌표계 변환(pyproj)            (지도·애니메이션)
 SGIS 인구/격자           격자 매칭·집계                  타임라인·주야 연출
 울산 인구현황(이동)       GeoJSON/Arrow export           테마(디자인 컨셉)
                              │                              │
                         [정적 데이터 저장]  ←───────────────┘
                         (S3 / 정적 호스팅 / 캐시)
```

설계 원칙: **무거운 좌표 변환·집계는 전처리(배치)에서 끝내고**, 프론트는 가공 완료된 경량 데이터(GeoJSON/Arrow)를 받아 렌더링·애니메이션에만 집중한다.

---

## 1. 프론트엔드 (시각화)

| 영역 | 선택 기술 | 역할 / 선정 이유 |
|---|---|---|
| 지도 렌더링 | **MapLibre GL JS** | 오픈소스 벡터 베이스맵, 다크/라이트/커스텀 스타일(디자인 컨셉 적용) |
| 대량 시각화 | **deck.gl** | WebGL 기반, 수십만 점/경로를 GPU로 렌더링 |
| 점 이동 애니메이션 | deck.gl **TripsLayer** | 시간축 기반 경로 이동·소멸(승하차 흐름) |
| 인구밀도 배경 | deck.gl **GridLayer / HeatmapLayer** | 균일 격자 밀도 히트맵 |
| 경로/노선 | deck.gl **PathLayer** | 노선별 색상 폴리라인 |
| 산점/정류장 | deck.gl **ScatterplotLayer** | 정류장·승객 점 |
| 주야 조명 연출 | deck.gl **LightingEffect / AmbientLight** | 일출~일몰 광량 보간 |
| UI 프레임워크 | **React** + Vite | 컴포넌트 기반, 타임라인·컨트롤 UI |
| 상태관리 | Zustand (경량) | 시간축·필터·테마 상태 |
| 차트(보조) | Chart.js | 시간대별 통행량 등 보조 그래프 |
| 좌표 변환(클라) | **proj4** | 필요 시 클라이언트 좌표 변환 |
| 스타일/테마 | CSS 변수 + 테마 토큰(JSON) | 디자인 컨셉 교체 가능 구조 |

> 빠른 프로토타입이 필요하면 **Kepler.gl**(코딩 최소화)로 PoC 후 deck.gl 커스텀으로 이관.

## 2. 데이터 파이프라인 (백엔드 / 전처리)

| 영역 | 선택 기술 | 역할 |
|---|---|---|
| 언어 | **Python 3.11+** | 수집·전처리 메인 |
| 데이터 처리 | **pandas** | 승하차·인구·이동 데이터 가공 |
| 대용량(옵션) | Polars / DuckDB | 대량 OD 레코드 고속 집계 |
| 좌표계 변환 | **pyproj** | WGS84(EPSG:4326) ↔ UTM-K(EPSG:5179) |
| 공간 연산 | **GeoPandas / Shapely** | 격자·집계구 매칭, 폴리곤 처리 |
| API 호출 | requests / httpx | 공공데이터 API 수집 |
| 출력 포맷 | **GeoJSON**, **Apache Arrow / Parquet** | 프론트 전달용 경량 포맷 |
| 스케줄링(옵션) | cron / Airflow | 정류장·노선 정보 주기 갱신 |

핵심 전처리 작업:
1. BIS 정류장 ID → 좌표 룩업 테이블 구축 (1회 수집·캐싱)
2. 국토부 교통카드 레코드의 승하차 정류장 → 좌표 매핑
3. GPS 좌표 → UTM-K 변환 → **격자 셀 매칭(floor 연산)** → 인구통계 조인
4. 격자 GeoJSON을 WGS84로 역변환(프론트 렌더용)
5. 시간(`RIDE_DT`~`GOFF_DT`) 기반 TripsLayer용 타임스탬프 경로 생성

## 3. 데이터 소스 / API

| 소스 | 포맷 | 용도 |
|---|---|---|
| 국토교통부 교통카드 합성데이터 | JSON/XML | 승하차 OD·인원·시간(이동 흐름 핵심) |
| 울산광역시 BIS | XML | 정류장 좌표·노선 형상·시간표 |
| SGIS 통계지도 API | JSON | 격자 인구밀도·집계구·지오코딩·좌표변환 |
| 행정안전부 행정동 성별·연령별 인구 | CSV/API | 연령·세대 정밀 인구 |
| 울산광역시 인구현황(전입·전출·순이동) | CSV | 행정동 순이동(차감 모드 기준값) |

인증: 공공데이터포털 일반 인증키(Encoding/Decoding), BIS 일일 트래픽 1,000회 → 정류장·노선 정보 캐싱 필수.

---

## 4. 좌표계 / 공간 표준

- 입력 GPS·웹지도: **WGS84 (EPSG:4326)**, 위경도
- 통계(격자·집계구): **UTM-K (EPSG:5179, GRS80)**, 미터
- 변환: 전처리는 `pyproj`, 클라이언트 보정은 `proj4`
- 격자 셀 매칭: `floor(X/셀크기)*셀크기` 산술 연산 (point-in-grid)
- 렌더 정렬: 격자 폴리곤을 WGS84로 역변환 후 deck.gl 표시

## 7. 기술 선택 요약 (한 줄 정리)

- **지도/시각화**: MapLibre GL + deck.gl (대량 점·경로 GPU 렌더링)
- **애니메이션**: TripsLayer로 하루 시간축 승하차 흐름 + 주야 조명 연출
- **데이터**: Python(pandas/GeoPandas) 전처리 → GeoJSON/Arrow 정적 서빙
- **좌표**: pyproj/proj4로 WGS84 ↔ UTM-K 변환, 격자 셀 매칭
- **테마**: 토큰 분리로 디자인 컨셉 교체 가능
- **배포**: 정적 호스팅 우선, 동적 필요 시 FastAPI 확장
