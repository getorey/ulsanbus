# 노선 정보 패널 설계서
ulsan_daynight_flow — 리스트에서 노선 선택 시 보여줄 운영 인사이트 패널

## 1. 목적
지금은 노선을 선택하면 경로·정류장만 그린다. 여기에 **그 노선의 운영 정보**(시간별 이용량, 배차 변화, 회전 시간, 구간 혼잡도, 정류장별 이용, 환승 연계)를 패널로 붙여 "노선 진단 카드"로 만든다.

## 2. 패널 UI
- 위치: 노선 선택 시 화면 우측에 슬라이드 인되는 **정보 패널**(폭 ~320px, 세로 스크롤). 미선택 시 숨김.
- 구성(위→아래):
  1. 헤더: 노선번호 · 분류(시내/마을) · 기점→종점
  2. 요약 카드: 일 이용 ○○명 · 정류장 ○개 · 총연장 ○km · 회전시간 ○분 · 평균 배차 ○분
  3. 시간별 이용량 막대(0~24시) — 하단 전체 히스토그램과 동일 스타일
  4. 시간대별 배차간격 선그래프(첨두 짧고 비첨두 길어지는 곡선)
  5. 구간별 재차인원/혼잡도 — 경로를 따라가는 미니 막대(또는 지도 경로선에 색 입힘)
  6. 정류장별 승차 Top 막대
  7. 환승 연계 Top(이 노선과 많이 갈아타는 노선/정류장)
- 지도 연동: 패널의 정류장/구간에 마우스를 올리면 지도에서 해당 위치 강조.

## 4. 데이터 확보 단계

### A. 지금 카드로 만드는 노선 사전집계(JSON)
노선별로 미리 집계해 `route_info_20260407.json`에 저장(패널이 즉시 로드):
```
route_info[ri] = {
  no, 기점, 종점, nStops, lenKm,
  hourBoard[24],            # 시간별 승차
  peakRatio, peakHour,
  topStops[ {name, board} ],
  segLoad[ {a,b,load} ],    # 구간 재차(카드 추정)
  estCycleMin              # 추정 회전시간 = lenKm / 22km/h * 2(왕복)
}
```
- lenKm·segLoad·topStops·hourBoard 모두 보유 데이터(card + route_stops)로 계산.
- estCycleMin은 "추정"으로 명시(브랜치: 정확값은 4-B).

### B. 신규 API 수집(보강) — 우선순위
1. **노선별 혼잡도(15142068)** + **노선별 차내 재차인원(15142063)**: 구간 혼잡도/재차의 정답값.
2. **TAGO 노선정보(배차간격·첫차·막차)** 또는 **울산 BIS 운행정보**: 시간대별 배차간격 + 운행시간(회전시간).
3. **노선별 경유정류장별 이용량(15142057)**: 정류장별 이용 정밀화.
4. **교통수단(15142085)**: 시내/마을 분류 라벨.
5. **환승통행량(15142049)**: 환승 연계.
- 수집 방식은 기존 download 스크립트와 동일(서비스키·시도코드 31·일자 20260407·rte_id 루프, 5/분·재시도·resume).

## 5. 핵심 계산식
- 총연장 lenKm = 경유정류장 폴리라인 누적거리.
- 회전시간(추정) estCycleMin = (lenKm ÷ 운행속도) × 60 × 2.  운행속도 기본 22km/h(카드 중앙값), 노선별 평균통행시간 API 있으면 대체.
- 배차간격(추정, API 없을 때) = 운행대수 미상 → 표시 보류. API 확보 시 시간대별 intervalTime 직접 표시.
- 구간 재차인원(카드 추정) = 그 구간을 지나는 통행 수(승차순번≤구간<하차순번). 단, 하차 보간 한계로 근사.

## 6. 구현 단계
- **1단계(즉시, 카드만)**: 패널 + 시간별 이용량·첨두·정류장Top·구간재차(추정)·회전시간(추정)·총연장. 추정값은 "추정" 뱃지 표기.
- **2단계(API 보강)**: 배차간격·회전시간(정확)·혼잡도·노선분류·환승을 위 API 수집 후 병합. 사전집계 JSON에 필드 추가.
- 성능: 노선 선택 시 해당 ri의 사전집계만 읽어 즉시 렌더(추가 연산 없음).
