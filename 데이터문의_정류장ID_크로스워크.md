# [문의] 교통카드 합성데이터 ↔ 울산 BIS 정류장·노선 ID 매핑(크로스워크) 자료 요청

안녕하세요. 공공데이터를 활용해 울산광역시 버스 인구이동(OD) 시각화를 진행 중인 이용자입니다.
두 개의 공공데이터를 결합하려는데 **정류장ID·노선ID 체계가 서로 달라 매칭이 되지 않아** 문의드립니다.
아래에 조회 출처, 매칭이 안 되는 항목, 실제로 뽑은 예시 데이터를 정리했습니다.

---

## 1. 사용한 데이터(조회 출처)

### (A) 국토교통부_지역별 교통카드이용 합성데이터  — 승하차 이동(OD) 데이터
- 공공데이터포털: https://www.data.go.kr/data/15154079/openapi.do
- End Point: `https://apis.data.go.kr/1613000/RegionalTransportationCardUsageSyntheticData`
- 오퍼레이션: `/getUlsanTransportationCardUsageSyntheticData`
- 조회 조건(예): `opr_ymd=20260101`, `rte_id=31001122`, `users_type_cd=01`, `ride_ctpv_cd=31`, `dataType=JSON`
- 사용 필드: `rte_id`(노선아이디), `ride_sttn_id`(승차정류장아이디), `goff_sttn_id`(하차정류장아이디)

### (B) 울산광역시 BIS 정보 — 정류장 좌표·노선 경로
- 공공데이터포털: https://www.data.go.kr/data/15052669/openapi.do
- End Point: `http://openapi.its.ulsan.kr/UlsanAPI/`
- 사용 오퍼레이션: `BusStopInfo.xo`(버스정류장정보), `RouteInfo.xo`(노선정보), `AllRouteDetailInfo.xo`(노선별정류장정보)
- 사용 필드: `STOPID`(정류장ID), `STOPX/STOPY`(경도/위도), `BRTID`(노선ID), `BRTNO`(노선번호)

### (참고) 정류소 위치 파일 — 좌표 보조
- 울산광역시_버스 정류소 위치 정보 (CSV): `서비스아이디` 필드
- 국토교통부_전국 버스정류장 위치정보 (CSV, https://www.data.go.kr/data/15067528 ): `정류장번호`, `모바일단축번호` 필드

---

## 2. 문의 사항(핵심)

**(A) 교통카드 합성데이터의 정류장ID/노선ID** 를 **(B) 울산 BIS의 정류장ID(STOPID)/노선ID(BRTID)** 또는
**정류장 좌표(위경도)** 와 연결할 수 있는 **매핑(크로스워크) 테이블 또는 코드 정의서**가 있는지 요청드립니다.

구체적으로 (A)의 `ride_sttn_id`, `goff_sttn_id`, `rte_id` 코드가
- 국가 표준 정류장ID/노선ID인지,
- 어떤 기준(표준)으로 부여된 값인지,
- (B) 울산 BIS의 STOPID/BRTID 또는 좌표와 어떻게 연결되는지

를 알 수 있는 자료(코드북/매핑표)가 필요합니다.

---

## 3. 매칭이 안 되는 지점 (확인 결과)

| 비교 대상 | (A) 교통카드 | (B) BIS / 위치파일 | 매칭 결과 |
|---|---|---|---|
| 정류장ID | `ride_sttn_id` 7자리 (예: 3101480) | BIS `STOPID` 9자리 (예: 140000184) | **0건 일치** |
| 정류장ID | `ride_sttn_id` 7자리 | 위치파일 `서비스아이디`/`모바일단축번호` 5자리 (예: 40117) | **0건 일치** |
| 노선ID | `rte_id` (예: 31001122) | BIS `BRTID` (예: 194000077) / `BRTNO`(111) | **0건 일치** |

> 참고: 위치파일 2종(울산 정류소 위치 `서비스아이디` ↔ 전국 위치 `모바일단축번호`)은 서로 3,913건 일치하여 **동일 체계**임을 확인했습니다.
> 즉 교통카드 합성데이터의 정류장ID/노선ID만 다른 체계로 보이며, 이를 좌표/BIS와 연결할 기준 자료가 필요합니다.

## 4. 실제 예시 데이터

### (A) 교통카드 합성데이터 응답 예시 (rte_id=31001122, opr_ymd=20260101, users_type_cd=01)

| ride_dt(승차일시) | ride_sttn_id(승차정류장) | goff_dt(하차일시) | goff_sttn_id(하차정류장) | rte_id(노선) |
|---|---|---|---|---|
| 20260101173209 | 3101480 | 20260101173606 | 3100144 | 31001122 |
| 20260101171107 | 3101442 | 20260101171249 | 3101441 | 31001122 |
| 20260101185200 | 3102956 | 20260101191827 | 3101267 | 31001122 |
| 20260101173950 | 3101446 | 20260101180730 | 3100439 | 31001122 |

→ 정류장ID가 모두 `31`로 시작하는 7자리(예: 3101480, 3100144).

### (B) 울산 BIS 버스정류장정보(BusStopInfo) 응답 예시

| STOPID(정류장ID) | STOPNAME(정류장명) | STOPX(경도) | STOPY(위도) |
|---|---|---|---|
| 140000184 | 노포동역(종점) | 129.09497371 | 35.28333284 |
| 140000185 | 노포동역(기점) | 129.09480702 | 35.28326138 |
| 140000203 | 금정체육공원입구 | 129.0986469 | 35.29170851 |

### (B) 울산 BIS 노선정보(RouteInfo) 응답 예시

| BRTID(노선ID) | BRTNO(노선번호) | COMPANY(회사) |
|---|---|---|
| 194000077 | 111 | 남성여객자동차(주) |
| 194000105 | 1114 | - |

→ BIS 정류장ID는 9자리(예: 140000184), 노선ID는 9자리(예: 194000077)로 (A)와 형식·값이 다릅니다.

---

## 5. 요청 자료(정리)

1. 교통카드 합성데이터의 `ride_sttn_id`/`goff_sttn_id`/`rte_id` **코드 정의(부여 기준)**
2. 위 정류장ID·노선ID ↔ **울산 BIS STOPID/BRTID** 매핑표, 또는
3. 위 정류장ID ↔ **정류장 좌표(위경도)** 매핑표(코드북)
4. (위 자료가 없을 경우) 정류장명 기준으로 매칭 가능한 참조 테이블 제공 여부

바쁘시겠지만 확인 부탁드립니다. 감사합니다.
